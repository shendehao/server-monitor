## 1. 项目概述

### 1.1 项目目标

构建一个集中式服务器管理平台，支持：

- **大屏展示** — 在大屏幕上实时展示所有服务器状态
- **实时监控** — CPU、内存、磁盘、网络等核心指标
- **告警通知** — 服务器异常时及时告警
- **历史查询** — 查看历史指标趋势

### 2.2 架构模式

采用 **前后端分离 + 单文件部署** 架构：

1. **Go 后端** — 负责 API、SSH 采集、WebSocket 推送、定时调度
2. **Vue 3 前端** — 负责大屏展示、图表渲染、用户交互
3. **Go embed** — 编译时将 Vue 静态文件嵌入 Go 二进制，最终只分发一个 `.exe`
4. **开发时分离** — Vue 独立 dev server + 代理到 Go 后端，热更新体验好

```
生产部署：
  server-monitor.exe (内含 Go 后端 + Vue 前端静态文件)
  └── data/monitor.db (SQLite 数据文件，自动创建)

开发模式：
  Go 后端 :8080  ←──代理──  Vue Dev Server :5173
```

## 4. 数据采集方案

### 4.1 采集方式：SSH 轮询（首选）

```
中控服务器 (Go) ──SSH──▶ 目标服务器 ──执行命令──▶ 返回指标数据
                         (每 10 秒一轮)
```

**采集流程：**

1. `robfig/cron` 触发定时任务
2. Go 启动 7 个 goroutine 并行 SSH 连接
3. 每个 goroutine 执行预定义的系统命令
4. 解析命令输出为结构化指标（Go struct）
5. 存入 `sync.Map` 内存缓存 + 写入 SQLite（GORM）
6. 通过 WebSocket 广播推送到所有前端客户端

**并发模型：**

```go
func (c *Collector) CollectAll() {
    var wg sync.WaitGroup
    for _, server := range c.servers {
        wg.Add(1)
        go func(s Server) {
            defer wg.Done()
            metrics, err := c.collectOne(s)
            if err != nil {
                c.handleOffline(s)
                return
            }
            c.cache.Store(s.ID, metrics)
            c.db.Save(metrics)
            c.hub.Broadcast(metrics)
        }(server)
    }
    wg.Wait()
}
```

### 4.2 采集指标

#### Linux 服务器

| 指标 | 采集命令 | 说明 |
|------|---------|------|
| CPU 使用率 | `top -bn1 \| grep 'Cpu(s)'` | 解析 idle 值计算使用率 |
| 内存使用率 | `free -m` | 解析 total/used/available |
| 磁盘使用率 | `df -h` | 各分区使用百分比 |
| 系统负载 | `cat /proc/loadavg` | 1/5/15 分钟负载 |
| 网络流量 | `cat /proc/net/dev` | 入站/出站字节数 |
| 运行时间 | `uptime -s` | 服务器启动时间 |
| 进程数 | `ps aux \| wc -l` | 运行中的进程总数 |

#### Windows 服务器

| 指标 | 采集命令 | 说明 |
|------|---------|------|
| CPU 使用率 | `wmic cpu get loadpercentage` | CPU 负载百分比 |
| 内存使用率 | `wmic OS get FreePhysicalMemory,TotalVisibleMemorySize` | 内存信息 |
| 磁盘使用率 | `wmic logicaldisk get size,freespace,caption` | 磁盘信息 |
| 运行时间 | `wmic os get lastbootuptime` | 启动时间 |
| 网络流量 | `netstat -e` | 网络统计 |

### 4.3 SSH 连接池

为避免频繁建立 SSH 连接，Go 后端维护 SSH 连接池：

```go
type SSHPool struct {
    mu      sync.RWMutex
    clients map[string]*ssh.Client  // serverID -> client
    config  map[string]*ssh.ClientConfig
}

func (p *SSHPool) Get(serverID string) (*ssh.Client, error) {
    p.mu.RLock()
    if client, ok := p.clients[serverID]; ok {
        // 检查连接是否存活
        if _, _, err := client.SendRequest("keepalive", true, nil); err == nil {
            p.mu.RUnlock()
            return client, nil
        }
    }
    p.mu.RUnlock()
    // 重新建立连接
    return p.reconnect(serverID)
}
```

### 4.4 后续演进：Agent 模式

当规模扩大或需要更高实时性时，可升级为 Agent 模式：

```
目标服务器 (Go Agent) ──HTTP POST──▶ 中控服务器 (接收端)
               定时 5 秒推送一次
```

Agent 为一个轻量 Go 二进制（约 5MB），无需安装运行时。

### 5.2 数据模型（Go Struct + GORM）

#### Server 模型

```go
type Server struct {
    ID        string    `gorm:"primaryKey" json:"id"`
    Name      string    `gorm:"not null" json:"name"`
    Host      string    `gorm:"not null" json:"host"`
    Port      int       `gorm:"default:22" json:"port"`
    Username  string    `gorm:"not null" json:"username"`
    AuthType  string    `gorm:"default:password" json:"authType"`  // password / key
    AuthValue string    `json:"-"`                                  // 不返回给前端
    OSType    string    `gorm:"default:linux" json:"osType"`       // linux / windows
    GroupName string    `json:"group"`
    SortOrder int       `gorm:"default:0" json:"sortOrder"`
    IsActive  bool      `gorm:"default:true" json:"isActive"`
    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}
```

#### Metric 模型

```go
type Metric struct {
    ID           uint      `gorm:"primaryKey" json:"id"`
    ServerID     string    `gorm:"index:idx_server_time" json:"serverId"`
    CPUUsage     float64   `json:"cpuUsage"`
    MemTotal     int64     `json:"memTotal"`     // MB
    MemUsed      int64     `json:"memUsed"`      // MB
    MemUsage     float64   `json:"memUsage"`
    DiskTotal    int64     `json:"diskTotal"`    // GB
    DiskUsed     int64     `json:"diskUsed"`     // GB
    DiskUsage    float64   `json:"diskUsage"`
    NetIn        int64     `json:"netIn"`        // bytes
    NetOut       int64     `json:"netOut"`       // bytes
    Load1m       float64   `json:"load1m"`
    Load5m       float64   `json:"load5m"`
    Load15m      float64   `json:"load15m"`
    ProcessCount int       `json:"processCount"`
    IsOnline     bool      `gorm:"default:true" json:"isOnline"`
    CollectedAt  time.Time `gorm:"index:idx_server_time" json:"collectedAt"`
}
```

#### Alert 模型

```go
type Alert struct {
    ID         uint       `gorm:"primaryKey" json:"id"`
    ServerID   string     `json:"serverId"`
    AlertType  string     `json:"alertType"`   // cpu_high / mem_high / offline / disk_full
    Message    string     `json:"message"`
    Severity   string     `gorm:"default:warning" json:"severity"` // info / warning / critical
    IsResolved bool       `gorm:"default:false" json:"isResolved"`
    CreatedAt  time.Time  `json:"createdAt"`
    ResolvedAt *time.Time `json:"resolvedAt"`
}
```

### 5.3 数据保留策略

### 7.3 ECharts 大屏特色效果

- **仪表盘 (gauge)** — 每个服务器卡片内嵌 CPU 仪表盘
- **渐变面积图 (area)** — CPU/内存趋势，带发光渐变效果

- **水球图 (liquidfill)** — 内存使用率，视觉冲击力强
- **动态数字翻牌** — 顶部统计数据动态变化
- **地图散点 (可选)** — 如果服务器分布在不同城市

```go
// 路由注册
func SetupRouter(r *gin.Engine) {
    api := r.Group("/api")

```

### 8.2 统一响应格式

```go
type Response struct {
    Success   bool        `json:"success"`
    Data      interface{} `json:"data,omitempty"`
    Error     string      `json:"error,omitempty"`
    Timestamp int64       `json:"timestamp"`
}

func OK(c *gin.Context, data interface{}) {
    c.JSON(200, Response{
        Success:   true,
        Data:      data,
        Timestamp: time.Now().Unix(),
    })
}

func Fail(c *gin.Context, code int, msg string) {
    c.JSON(code, Response{
        Success:   false,
        Error:     msg,
        Timestamp: time.Now().Unix(),
    })
}
```

## 9. 告警规则

### 9.1 默认告警阈值

| 指标 | 警告阈值 | 危险阈值 | 触发条件 |
|------|---------|---------|---------|
| CPU 使用率 | > 80% | > 95% | 连续 3 次采集超标 |
| 内存使用率 | > 80% | > 95% | 连续 3 次采集超标 |
| 磁盘使用率 | > 80% | > 95% | 单次超标 |
| 服务器离线 | - | 离线 | SSH 连接失败 2 次 |
| 系统负载 | > CPU核数 | > CPU核数×2 | 连续 3 次超标 |

## 12. 构建与部署

### 12.1 Makefile

```makefile
# 开发模式
dev-server:
	cd server && go run main.go

dev-frontend:
	cd frontend && npm run dev

# 构建生产版本
build:
	cd frontend && npm run build
	cd server && CGO_ENABLED=1 go build -o ../dist/server-monitor.exe .

# Windows 交叉编译
build-windows:
	cd frontend && npm run build
	cd server && GOOS=windows GOARCH=amd64 CGO_ENABLED=1 go build -o ../dist/server-monitor.exe .

# 清理
clean:
	rm -rf dist/ frontend/dist/
```

### 12.2 Go embed 嵌入前端

```go
// server/embed.go
package main

import (
    "embed"
    "io/fs"
    "net/http"
)

//go:embed all:../frontend/dist
var frontendFS embed.FS

func staticHandler() http.FileSystem {
    sub, _ := fs.Sub(frontendFS, "frontend/dist")
    return http.FS(sub)
}
```

```bash
# 生产部署只需：
```

- **Go** >= 1.22（仅构建时需要）

## 14. 安全考虑

1. **SSH 凭证加密** — 数据库中的密码使用 AES 加密存储，运行时解密
2. **访问控制** — 可选 JWT 登录认证（Gin 中间件）
3. **网络隔离** — 建议中控服务器与目标服务器在同一内网
4. **CORS 配置** — 生产环境限制允许的域名
5. **日志脱敏** — zap logger 不打印密码和密钥
6. **HTTPS** — 建议通过 Nginx 反向代理提供 HTTPS
7. **横向移动** — 支持内网扫描发现存活主机（TCP 端口探测 445/135/5985/3389/22），通过 WMI 或 WinRM 远程执行 PowerShell cradle 将 Agent 部署到同网段 Windows 主机，需提供目标凭据，管理端控制触发
